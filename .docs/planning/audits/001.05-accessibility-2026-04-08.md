# Audit Report — Accessibility
**Run**: 001.05 | **Date**: 2026-04-08 | **Score**: 57 / 100 (D)

## Summary
The app has some accessibility groundwork in place: nav has `aria-label`, hamburger/close buttons have `aria-label`, the search input has `aria-label`, form fields on the signin page use `htmlFor` labels. However, several critical gaps exist: no skip navigation link anywhere in the shell, no modal focus trapping for any of the inline modals (directory-tree, ideas, settings panels), and the whiteboard/schema canvas pages are entirely inaccessible to keyboard-only or screen reader users. Inline style `outline: none` on `.nb-input:focus` removes the default focus ring without a visible replacement for that class. Touch targets on some tool buttons are below 44×44px.

## Recommendations
1. Add a visually-hidden skip link as the first focusable element in `app-shell.tsx` that jumps to `<main>` via an id.
2. Implement focus lock (e.g., `focus-trap-react`) on all inline modals in directory-tree, ideas, and kanban pages.
3. Ensure zoom/toolbar buttons meet 44×44px minimum. Increase the 28×28 whiteboard buttons and 16×16 entity card buttons.
