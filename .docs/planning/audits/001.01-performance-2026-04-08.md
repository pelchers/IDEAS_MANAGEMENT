# Audit Report — Performance
**Run**: 001.01 | **Date**: 2026-04-08 | **Score**: 55 / 100 (D)

## Summary
The app uses a client-heavy architecture with 15 pages annotated `"use client"`, no React Query (raw `fetch` + `useState`), no `next/image` optimisation, and two monolithic pages (whiteboard: 2,336 lines; schema: 2,853 lines) that ship enormous JS bundles. Google Fonts are loaded with `display=swap` via external link in `globals.css`, which is acceptable but not optimal. No lazy code-splitting exists for large sub-sections. These issues combine to create a material risk of slow initial load, large bundles, and excessive re-renders.

### HIGH

| ID | Severity | File | Finding |
|---|---|---|---|
| P-02 | High | All authenticated pages | No React Query, SWR, or any caching layer — every page uses raw `fetch` + `useState`. Data is re-fetched on every mount with no deduplication, stale-while-revalidate, or background refresh. This is a staleTime violation (check requires >60s). |
| P-03 | High | `src/app/(authenticated)/projects/[id]/whiteboard/page.tsx` | 10 `useEffect` hooks and 2,336 lines with no `React.memo`, `useMemo` on computed values, or component extraction. Canvas redraw triggered on any state change in the parent. |
| P-04 | High | `src/components/shell/app-shell.tsx` | No lazy loading for below-fold or heavy sub-components (AiHelper, SyncStatusIndicator always mounted, no `next/dynamic`). AiHelper with its full AI chat panel is loaded on every authenticated page regardless of usage. |

### MEDIUM

| ID | Severity | File | Finding |
|---|---|---|---|
| P-05 | Medium | `src/app/globals.css` | Fonts loaded via Google Fonts CSS import (`@import url(...)`) instead of `next/font`. Eliminates build-time font optimisation, render-blocking risk on first paint. |
| P-06 | Medium | `next.config.ts` | `nextConfig` is an empty object — no `images.domains`, no bundle analyser, no `experimental.optimizeCss`, no output caching headers configured. |
| P-07 | Medium | `src/app/(authenticated)/projects/[id]/schema/page.tsx`, `whiteboard/page.tsx` | 10 useEffect hooks per page with no dependency arrays reviewed for over-firing. Schema page has its own 10 useEffect calls including canvas redraws. |

### LOW

| ID | Severity | File | Finding |
|---|---|---|---|
| P-08 | Low | All pages | No `<img>` elements found using native img instead of `next/image` (only canvas-embedded images), but MediaItem URLs stored as base64 dataURLs in DB — large media items will bloat artifact JSON payloads. |
| P-09 | Low | `src/app/globals.css` | Only one breakpoint defined (`@media (max-width: 480px)`) — no tablet (768px) or intermediate breakpoints in global styles. |

## Recommendations
1. Extract whiteboard and schema canvas logic into `next/dynamic`-loaded modules to enable route-level code splitting.
2. Add React Query (or SWR) as a shared data-fetching layer with a default `staleTime` of at least 60 seconds.
3. Replace the Google Fonts CSS `@import` with `next/font/google` for build-time font optimisation.
