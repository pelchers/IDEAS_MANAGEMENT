## Summary
The app has a health endpoint (`/api/health`) that checks DB connectivity and returns proper 200/503 status codes. Environment variables are documented in `.env.example` with clear comments. ## Recommendations
1. Create a basic GitHub Actions workflow that runs `pnpm lint`, `pnpm typecheck`, `pnpm test`, and `pnpm build` on every pull request.
2. Install and configure Sentry (`@sentry/nextjs`) with environment-specific DSNs for production error tracking.
3. Add a `docker-compose.yml` at the root that starts a PostgreSQL container, enabling zero-friction local development onboarding.
