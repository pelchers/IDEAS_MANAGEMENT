# Audit Report — SEO
**Run**: 001.03 | **Date**: 2026-04-08 | **Score**: 53 / 100 (D)

## Summary
The app is a fully authenticated SaaS tool — the vast majority of content is behind auth and does not need SEO. However, the public-facing sign-in, sign-up, and landing pages lack proper metadata, Open Graph, and Twitter Card tags. There is no sitemap.xml, no robots.txt, no canonical URL system, and no structured data. The root layout provides only a basic title/description. For a product-level app heading toward public launch, these gaps matter for discoverability of the public pages and for link sharing.

## Findings

### HIGH

| ID | Severity | File | Finding |
|---|---|---|---|
| E-01 | High | `src/app/layout.tsx` | No Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`, `og:type`) in the root layout metadata. Without OG tags, link unfurling in Slack, Discord, Twitter, and iMessage shows a blank or incomplete card. |
| E-02 | High | `public/` (missing) | No `robots.txt` and no `sitemap.xml` exist anywhere in the project. Authenticated routes should be blocked via robots.txt; the marketing/auth pages (signin, signup) should be in the sitemap. |

### MEDIUM

| ID | Severity | File | Finding |
|---|---|---|---|
| E-03 | Medium | `src/app/signin/page.tsx`, `src/app/signup/page.tsx` | No `generateMetadata` or page-level metadata exports. Pages are indexed with the root layout's generic title "Idea Management" instead of page-specific titles like "Sign In — Idea Management". |
| E-04 | Medium | `src/app/layout.tsx:4-7` | No Twitter Card tags (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`). Root metadata only sets title and description. |
| E-05 | Medium | All pages | No canonical URL meta tag configured anywhere. If the app is accessible from multiple domains or with/without www, duplicate content issues will arise. |
| E-06 | Medium | `src/app/page.tsx` | Root page immediately redirects to `/dashboard`. It has no metadata, no content, and no canonical. While authenticated users are expected to hit this, unauthenticated users need to be redirected to a proper marketing/landing page or at minimum the signin page, not directly to `/dashboard` which would then redirect back. |

### LOW

| ID | Severity | File | Finding |
|---|---|---|---|
| E-07 | Low | `src/app/layout.tsx` | The root metadata description ("Manage ideas, projects, kanban boards, whiteboards, and more.") is 63 characters — within the 120-160 char optimal range? No, it is 63 chars — below the recommended minimum of 120 chars for best SERP display. |
| E-08 | Low | All authenticated pages | Pages behind auth do not need SEO, but they also lack `<meta name="robots" content="noindex">` tags — an issue if the app is ever accidentally crawled after a misconfigured robots.txt. |
| E-09 | Low | All pages | No structured data (JSON-LD) on any public page. For a SaaS tool, WebApplication schema on the landing/signin page would improve rich results. |

## Recommendations
1. Add a `robots.txt` at `public/robots.txt` disallowing `/dashboard`, `/projects`, `/ai`, `/settings`, `/profile` and allowing the sign-in/sign-up pages.
2. Add Open Graph and Twitter Card tags to the root layout metadata and page-specific metadata to sign-in and sign-up pages.
3. Create a `sitemap.xml` or Next.js `app/sitemap.ts` that lists the public-facing pages.
