# Audit Report — UX
**Run**: 001.09 | **Date**: 2026-04-08 | **Score**: 68 / 100 (C)

## Summary
The app has a strong, consistent Brutalist/neo-brutalist visual language applied across most pages. Loading states are present on all data-fetching pages (dashboard, projects, profile, conflicts). Empty states are implemented for projects (with helpful CTA) and AI sessions. Error boundaries exist (Next.js `error.tsx`) and page-level error states are shown. The main UX gaps are: no toast/notification system for async operations (form saves, project creation, AI tool executions show no feedback), no dedicated 404 page, `window.confirm()` dialogs used for destructive action confirmation (low quality UX), search input in the top bar is non-functional (no handler connected), and no back-button preservation of state on canvas pages.

Note: Live checks (Playwright) were SKIPPED — dev server not running during audit.

### MEDIUM

| ID | Severity | File | Finding |
|---|---|---|---|
| UX-03 | Medium | `src/components/shell/app-shell.tsx:242-256` | The search input in the top bar has no `onChange` handler, no `onSubmit`, and no routing behaviour. It is purely cosmetic — typing into it does nothing. This creates a deceptive UX where users expect global search functionality that doesn't exist. |
| UX-04 | Medium | `src/app/(authenticated)/ai/page.tsx:149,156`, `projects/page.tsx:139` | `window.confirm()` is used for destructive action confirmation (clear chat session, delete all sessions, delete project). Native `window.confirm` dialogs are visually inconsistent with the branded UI, cannot be styled, and on mobile are often blocked by browsers. Should be replaced with inline confirmation UI. |
| UX-05 | Medium | `src/app/(authenticated)/projects/[id]/whiteboard/page.tsx`, `schema/page.tsx` | Canvas state (zoom, pan position, selected tools) is stored in component state and lost on navigation away. Back button or refresh resets the canvas to default position. Users lose context when switching between pages. |
| UX-06 | Medium | `src/components/shell/app-shell.tsx` | No responsive layout for the navigation drawer on mobile. The drawer is 280px fixed width with no mobile-specific adaptations. The main content at `padding: "clamp(16px, 2.5vw, 48px)"` will be extremely cramped on 390px viewports when the drawer is open. |

## Recommendations
1. Install and configure a toast library (e.g., `sonner` or `react-hot-toast`) and add success/error notifications to all mutation operations: project create/update/delete, artifact save, AI tool execution.
2. Create `src/app/not-found.tsx` with a branded 404 page that links back to the dashboard.
3. Replace all `window.confirm()` calls with an inline confirmation component that matches the brutalist design system.
