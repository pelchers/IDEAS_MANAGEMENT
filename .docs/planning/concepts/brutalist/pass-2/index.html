WORKING DOCUMENT

§1 — PROJECT OVERVIEW

Platform Core encompasses all foundational services required for the idea management system. This includes user authentication (OAuth2 + JWT), API gateway routing, rate limiting, and session management. The project is currently in its third sprint cycle with 12 active tasks distributed across 3 team members.

§2 — CURRENT OBJECTIVES

  • Migrate authentication from session-based to JWT token flow
  • Implement rate limiting middleware (100 req/min default)
  • Add WebSocket support for real-time notifications
  • Set up distributed caching layer with Redis

§3 — NOTES & ANNOTATIONS

NOTE

Database migration scripts must be run before deploying auth changes. See p.20 for schema details.

CAUTION

Rate limiter currently has a known issue with WebSocket connections. Ticket #247 tracks this.
