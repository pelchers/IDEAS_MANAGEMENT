# Deployment and Hosting

## Mobile Client

- distributed as a React Native + Expo app
- likely requires platform-native builds if terminal capabilities outgrow Expo Go constraints

## PC Host

- relies on the user's existing PC and SSH availability
- may optionally ship a lightweight host companion for repo indexing and structured metadata

## Operational Model

This idea does not require a cloud backend to prove the core workflow. The product can start as a direct device-to-PC model, then add cloud-backed account services later if synchronization, relay, or remote access outside local networking is desired.
